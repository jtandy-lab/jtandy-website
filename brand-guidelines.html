02 — Core Attributes

Five Lenses

Every visual element is filtered through these five attributes. If it doesn't serve one — remove it.

Attribute Visual Translation
Control Structured grids, clean alignment
Engineering Subtle system diagrams, process visuals
Trust Conservative colour palette
Commercial Large numeric emphasis
Clarity White space + minimal decoration
If a design element doesn't support one of these five attributes — remove it.

06 — Imagery Guidelines

Visual Content

Imagery must be practical and reinforce the infrastructure positioning.

✓ Use
  • Real website screenshots
  • Before / After comparisons
  • Annotated structure visuals
  • Simple flow diagrams
  • Device mockups (minimal)
✕ Do Not Use
  • Stock tech imagery
  • Abstract AI graphics
  • Neon gradients
  • Floating blobs
  • Futuristic robots
07 — Iconography

Icon Style

Icons are structural support — not decoration.

✓ Style
  • Thin line icons
  • Single stroke weight
  • Consistent sizing
  • Structural support only
✕ Avoid
  • Filled cartoon icons
  • Emoji icons
  • Decorative illustration
  • Multi-colour icons
08 — Motion & Interaction

Minimal Motion

Motion should be felt, not seen. Functional only.

✓ Allowed
  • 150–200ms hover transitions
  • Subtle shadow increase on hover
  • Button press translateY(1px)
✕ Not Allowed
  • Parallax
  • Big fade animations
  • Scroll gimmicks
  • Floating animations

11 — Dark / Light Strategy

When to Use Each

Grey is reserved — not the default. White is default for most pages.

Grey — Reserved For

  • Hero of flagship pages
  • Revenue Engine page
  • Final CTA sections

White — Default For

  • Process pages
  • Pricing
  • FAQ
  • Audit
Contrast between pages creates depth and intentionality. Don't make everything grey.
