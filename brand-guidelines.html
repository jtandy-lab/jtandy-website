01 — Brand Positioning

Know What You Are

Before any visual decisions are made, this must be clear. Every design choice flows from this anchor.

You Are Not

02 — Core Attributes

These Drive Design Decisions

Every visual choice maps to one of these attributes. If a design element doesn't support one, remove it.

Attribute Visual Translation
Control Structured grids, clean alignment
Engineering Subtle system diagrams, process visuals
Trust Conservative colour palette
Commercial Large numeric emphasis
Clarity White space + minimal decoration
If a design element doesn't support one of these five attributes — remove it.

06 — Imagery Guidelines

Practical, Not Conceptual

Imagery must feel practical. Not conceptual. Every image should reinforce infrastructure positioning.

✓ Use
  • Real website screenshots
  • Before / After comparisons
  • Annotated structure visuals
  • Simple flow diagrams
  • Device mockups (minimal)
✕ Do Not Use
  • Stock tech imagery
  • Abstract AI graphics
  • Neon gradients
  • Floating blobs
  • Futuristic robots
07 — Iconography

Icons Support Structure

Icons are functional, not decorative.

✓ Style
  • Thin line icons
  • Single stroke weight
  • Consistent sizing
  • Structural support only
✕ Avoid
  • Filled cartoon icons
  • Emoji icons
  • Decorative illustration
  • Multi-colour icons
08 — Motion & Interaction

Keep Motion Minimal

You are infrastructure, not a startup demo.

✓ Allowed
  • 150–200ms hover transitions
  • Subtle shadow increase on hover
  • Button press translateY(1px)
✕ Not Allowed
  • Parallax
  • Big fade animations
  • Scroll gimmicks
  • Floating animations

11 — Dark vs Light Strategy

Contrast Creates Intentionality

Dark navy is reserved — not the default. White is default for most pages.

Dark Navy — Reserved For
  • Hero of flagship pages
  • Revenue Engine page
  • Final CTA sections
White — Default For
  • Process pages
  • Pricing
  • FAQ
  • Audit
Contrast between pages creates depth and intentionality. Don't make everything dark.
